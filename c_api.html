Forward projection using the Joseph 3D algorithm.

Note

All pointers can be host pointers, CUDA device pointers, or CUDA managed pointers.

Parameters:

  • xstart – Pointer to array of shape [3*nlors] with the coordinates of the start points of the LORs. The start coordinates of the n-th LOR are at xstart[n*3 + i] with i = 0,1,2. Units are the ones of voxsize.

  • xend – Pointer to array of shape [3*nlors] with the coordinates of the end points of the LORs. The end coordinates of the n-th LOR are at xend[n*3 + i] with i = 0,1,2. Units are the ones of voxsize.

  • img – Pointer to array of shape [n0*n1*n2] containing the 3D image used for forward projection. The pixel [i,j,k] is stored at [n1*n2*i + n2*j + k].

  • img_origin – Pointer to array [x0_0, x0_1, x0_2] of coordinates of the center of the [0,0,0] voxel.

  • voxsize – Pointer to array [vs0, vs1, vs2] of the voxel sizes.

  • p – Pointer to array of length nlors where the forward projection results will be stored.

  • nlors – Number of geometrical LORs.

  • img_dim – Pointer to array with dimensions of the image [n0, n1, n2].

  • device_id – ID of the device to use for computation (default: 0).

  • threadsperblock – Number of threads per block for GPU computation (default: 64).

3D sinogram TOF Joseph forward projector

Note

All pointers can be host pointers, CUDA device pointers, or CUDA managed pointers.

Parameters:

  • xstart – pointer to array of shape [3*nlors] with the coordinates of the start points of the LORs. The start coordinates of the n-th LOR are at xstart[n*3 + i] with i = 0,1,2. Units are the ones of voxsize.

  • xend – pointer array of shape [3*nlors] with the coordinates of the end points of the LORs. The end coordinates of the n-th LOR are at xend[n*3 + i] with i = 0,1,2. Units are the ones of voxsize.

  • img – pointer array of shape [n0*n1*n2] containing the 3D image to be projected. The voxel [i,j,k] is stored at index n1*n2*i + n2*j + k.

  • img_origin – pointer array [x0_0,x0_1,x0_2] of coordinates of the center of the [0,0,0] voxel.

  • voxsize – pointer array [vs0, vs1, vs2] of the voxel sizes.

  • p – pointer to array of length nlors*n_tofbins (output) used to store the projections. The ordering is row-major per LOR: [LOR0-TOFBIN-0, LOR0-TOFBIN-1, … LOR0-TOFBIN-(n-1), LOR1-TOFBIN-0, LOR1-TOFBIN-1, … LOR1-TOFBIN-(n-1), … LOR(N-1)-TOFBIN-0, LOR(N-1)-TOFBIN-1, … LOR(N-1)-TOFBIN-(n-1)]

  • nlors – number of geometrical LORs

  • img_dim – array with dimensions of image [n0,n1,n2]

  • tofbin_width – width of the TOF bins in spatial units (units of xstart and xend)

  • sigma_tof – pointer to array of length 1 or nlors (depending on lor_dependent_sigma_tof) with the TOF resolution (sigma) for each LOR in spatial units (units of xstart and xend)

  • tofcenter_offset – pointer to array of length 1 or nlors (depending on lor_dependent_tofcenter_offset) with the offset of the central TOF bin from the midpoint of each LOR in spatial units. A positive value means a shift towards the end point of the LOR.

  • n_sigmas – number of sigmas to consider for calculation of TOF kernel

  • n_tofbins – number of TOF bins

  • lor_dependent_sigma_tof – unsigned char 0 or 1 0 means that the first value in the sigma_tof array is used for all LORs 1 (non-zero) means that the TOF resolutions are LOR dependent

  • lor_dependent_tofcenter_offset – unsigned char 0 or 1 0 means that the first value in the tofcenter_offset array is used for all LORs 1 (non-zero) means that the TOF center offsets are LOR dependent

  • device_id – ID of the device to use for computation (default: 0).

  • threadsperblock – Number of threads per block for GPU computation (default: 64).

void joseph3d_tof_sino_back(const float *xstart, const float *xend, float *img, const float *img_origin, const float *voxsize, const float *p, size_t nlors, const int *img_dim, float tofbin_width, const float *sigma_tof, const float *tofcenter_offset, float n_sigmas, short n_tofbins, unsigned char lor_dependent_sigma_tof, unsigned char lor_dependent_tofcenter_offset, int device_id = 0, int threadsperblock = 64)¶

TOF sinogram backprojection using the Joseph 3D algorithm.

The function backprojects a TOF sinogram into a 3D image volume using the Joseph ray-driven algorithm. The projection data p is organized row-major per LOR: [LOR0-TOFBIN-0, LOR0-TOFBIN-1, …, LOR0-TOFBIN-(n_tofbins-1), LOR1-TOFBIN-0, LOR1-TOFBIN-1, …, LOR1-TOFBIN-(n_tofbins-1), …]. Values from p are distributed into img (accumulated, not overwritten).

Note

All pointers can be host pointers, CUDA device pointers, or CUDA managed pointers.

Parameters:

  • xstart – Pointer to array of shape [3*nlors] with start coordinates for each LOR (xstart[n*3 + i], i=0..2). Units are those of voxsize.

  • xend – Pointer to array of shape [3*nlors] with end coordinates for each LOR (xend[n*3 + i], i=0..2). Units are those of voxsize.

  • img – Pointer to array of shape [n0*n1*n2] containing the 3D image to add backprojected contributions into. The element (i,j,k) is stored at index n1*n2*i + n2*j + k. Values are added to the existing contents of this array.

  • img_origin – Pointer to array [x0_0, x0_1, x0_2] giving the coordinates of the center of the voxel at index [0,0,0].

  • voxsize – Pointer to array [vs0, vs1, vs2] specifying voxel sizes in the same units as LOR coords.

  • p – Pointer to TOF sinogram data of length nlors * n_tofbins (see details).

  • nlors – Number of geometric LORs.

  • img_dim – Pointer to array [n0, n1, n2] with image dimensions. Can be host/device/managed.

  • tofbin_width – Width of each TOF bin in spatial units (same units as LOR coordinates).

  • sigma_tof – Pointer to array of length 1 or nlors (depending on lor_dependent_sigma_tof) specifying TOF sigma(s) in spatial units.

  • tofcenter_offset – Pointer to array of length 1 or nlors (depending on lor_dependent_tofcenter_offset) specifying per-LOR offset of the central TOF bin from the geometric midpoint (positive towards xend).

  • n_sigmas – Number of sigmas to consider when evaluating the TOF kernel (controls kernel radius).

  • n_tofbins – Number of TOF bins per LOR.

  • lor_dependent_sigma_tof – If non-zero, sigma_tof contains one sigma per LOR; otherwise the first element is used for all LORs.

  • lor_dependent_tofcenter_offset – If non-zero, tofcenter_offset contains one offset per LOR; otherwise the first element is used for all LORs.

  • device_id – CUDA device to use (default: 0). If negative, CPU path is used when available.

  • threadsperblock – Number of CUDA threads per block for GPU execution (default: 64).

Returns:
